# 🔐 Security Policy

## 📌 Supported Versions

This project is actively maintained. Security updates will be applied to the latest version.

| Version        | Supported |
| -------------- | --------- |
| Latest         | ✅ Yes     |
| Older Versions | ❌ No      |

---

## 🚨 Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly.

### 📩 How to Report

* Email: chetansonigara01@gmail.com
* Or open a **private issue** (if supported)

---

## ⚠️ Please Do NOT

* Do not publicly disclose vulnerabilities before reporting
* Do not exploit vulnerabilities beyond proof of concept

---

## 🛠️ What to Include

When reporting, please provide:

* Description of the vulnerability
* Steps to reproduce
* Potential impact
* Suggested fix (if possible)

---

## ⏱️ Response Time

We aim to:

* Acknowledge within **48 hours**
* Investigate within **3–5 days**
* Resolve as soon as possible

---

## 🔒 Security Best Practices

For contributors and users:

* Keep dependencies updated
* Avoid hardcoding sensitive information
* Use secure coding practices
* Validate all inputs

---

## 📦 Dependencies

We rely on open-source libraries.
If a vulnerability is found in dependencies:

* We will update to a secure version
* Patch the issue as soon as possible

---

## ⚖️ Responsible Disclosure

We appreciate responsible disclosure and will:

* Investigate all valid reports
* Credit reporters (if desired)
* Work transparently on fixes

---

## 🙏 Acknowledgment

Thank you for helping keep this project secure! 🔐

---

# 🚀 Commitment

We are committed to maintaining a **secure, reliable, and trustworthy project** for all users.
